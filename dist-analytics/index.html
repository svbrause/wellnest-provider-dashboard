Discussed treatments — analytics

Explore patient exports (CSV with JSON in Treatments Discussed). Charts cover service staff, treatments, timelines, products, and regions. Everything runs locally in your browser; no data is uploaded.

Over HTTP, the page tries to load bundled data (patients-discussed-data.csv next to this file or at the site root), then your last session (same browser), then the dev sample CSV. Add ?nodata=1 to skip auto-load. Use ?data=/your-export.csv for a specific file (same origin). Vercel: after npm run build, the analytics HTML and public/patients-discussed-data.csv are in dist/; replace that CSV to change the default dataset. Opening as file:// still needs the file picker unless you use localStorage from a prior load. Deploy: with the main app, npm run build → open /patients-discussed-analytics.html on your domain. For a tiny site that is only this tool at /, run npm run build:analytics then npx vercel dist-analytics --prod.

Video production priorities

Suggested order for patient-education clips. A generic “Neurotoxin” video is weak next to something that matches how people actually talk in the chart — e.g. area (glabella, prejowl sulcus) and/or product / line detail from each JSON line. Use Topic granularity to choose abstraction: broad family vs treatment + regions (comma tokens from the region field) vs treatment + product (same as matrix “detail” rows). “One row per area” modes split multi-area lines so each site surfaces its own priority (better for hyper-targeted assets; may duplicate the same consult across topics). Ranking is driven mainly by Add next visit and Wishlist lines (pre-visit + nurture). Now, skincare, empty / other timelines are down-weighted so they rarely float a topic to the top unless volume is huge. Recency from addedAt still nudges fresh discussions. Brand stays in titles when present. Each ranked row lists common brands and product/type lines from the underlying JSON (comma-split products, brand field) plus an example angle like “PRFM — microneedling” when data supports it. Still blends staff/patient breadth and product spread. Heuristic only — align with clinical and marketing judgment.
